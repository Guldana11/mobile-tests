# BACKLOG — покрытие автотестами

Бэклог на покрытие **авторизованной зоны** приложения автотестами.
Каркас (DriverFactory, BaseTest, retry, cross-platform Android/iOS) и воронка до входа
(язык → приветствие → филиалы/о банке → телефон → пароль → PIN → главный экран) уже закрыты.

Приоритеты: `P0` — критично/фундамент, `P1` — основной функционал, `P2` — важно, но не блокирует.
Оценки в рабочих днях, «грязные» (одним инженером).

## EPIC 7. Безопасность `[P1, сквозной]`

Client-side / UI security controls авторизованной зоны. Прогоняются на тестовых аккаунтах в dev-окружении.

| ID | Задача | UI-автотест | Оценка | Статус |
|----|--------|-------------|--------|--------|
| SEC-3 | Перевод нельзя завершить без OTP (SMS) | ✅ в `SecurityTest` | — | **готов (iOS зелёный)** |
| SEC-4 | Маскирование пароля + нет утечки plaintext в a11y-дерево | ✅ в `SecurityTest` (2 кейса) | — | **готов (iOS зелёный)** |
| SEC-1 | Brute-force пароля → блокировка входа («Вход заблокирован до…») | ⚠️ нужен **отдельный расходный аккаунт** + opt-in группа (блокирует аккаунт!) | ~0.5д | **отложен** (нет расходного аккаунта) |
| SEC-2 | Brute-force PIN → выход/блокировка | ⚠️ как SEC-1 | ~0.5д | tracked |
| SEC-5 | Re-auth (PIN) при возврате приложения из фона | хелпер `background app` | ~0.5д | tracked |
| SEC-6 | Нет восстановления сессии после force-kill (снова логин/PIN) | reinstall/restart хелпер есть | ~0.5д | tracked |
| SEC-7 | FLAG_SECURE: скриншот на чувствительных экранах запрещён (Android) | анализ `getScreenshotAs` | ~0.5д | tracked |
| SEC-8 | Deeplink/intent не обходит авторизацию (Android) | intent-запуск | ~1д | tracked |
| SEC-9 | Анти-enumeration: ошибка входа не раскрывает существование номера | сравнение текстов ошибок (учесть лимит попыток) | ~0.5д | tracked |

**Вне Appium (отдельный инструментарий, рекомендации в `REPORT.md`):** SSL-pinning/MITM (mitmproxy/Burp),
статика/секреты в APK·IPA и root-bypass (MobSF), jailbreak/root-detection (рутованный девайс),
backend authz / IDOR (API-тесты).

**DoD:** `SecurityTest` зелёный на Android+iOS; деструктивные кейсы (SEC-1/2) изолированы в opt-in группу
с выделенным аккаунтом; рекомендации по не-UI проверкам переданы команде приложения.

## Итог

- **Всего:** ~30–35 рабочих дней (≈6–7 недель одним инженером).
- **P0 (EPIC 0–1):** ~2 недели до рабочего smoke авторизованной зоны.

### Известные ограничения окружения
- iOS: часть кейсов скипается из-за багов приложения (phone-back, recovery) — паритет ≈ ×1.3–1.5, не ×2.
- Android: full-reset флейки + деградация эмулятора на длинных прогонах (см. T-25).
